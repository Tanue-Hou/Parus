# Parus 词库多源融合策略

> 本文档定义俄语词条的多源数据融合规则。
> 执行者：LLM 智能体 + Python 数据管线。
> 核心原则：多源输入 → 单一最优输出，不标注来源，直接融合。

---

## 一、融合对象

每个俄语词条从多个数据源收集原始数据，融合为一条完整记录。

### 输入源

| 数据源 | 提供什么 |
|--------|----------|
| BKRS | 中文释义、内嵌例句、词性(部分) |
| OpenRussian | 英文翻译、变格变位(完整)、词频排名、词性、性/体 |
| Kaikki/Wiktionary | 英文释义、变格变位、俄语单语释义、词源、同义反义、例句 |
| Tatoeba | 中俄双语例句对 |
| pymorphy3 | 形态分析(词形还原)、变格生成 |

### 输出结构

每个词条最终产出一条记录，包含以下字段：

```
lemma            — 去重音小写原形 (搜索用)
lemma_stressed   — 带重音原形 (展示用)
pos              — 词性
frequency        — 词频排名
conjugation_type — 动词变位类型
definition       — 融合后的中文释义 (唯一一条，不标注来源)
inflections[]    — 融合后的变格变位表 (去重，规范化tag)
examples[]       — 融合后的例句 (去重，按质量排序)
etymology        — 词源
synonyms         — 同义词
antonyms         — 反义词
difficulty       — 难度等级
```

## 二、释义融合规则

### 2.1 核心原则

将所有来源的中文释义、英文释义、俄语释义汇总后，由 LLM 进行语义级融合，输出**一条**中文释义。释义中不出现"〔BKRS〕""〔Kaikki〕"等来源标记。

### 2.2 融合算法

输入：
```
中文释义: [BKRS] "1) 水 2) 水面 3) 水流 4) 矿泉 5) 空话"
英文翻译: [OpenRussian] "water, aqua"
英文释义: [Kaikki] "water, moisture, waterway, body of water"
俄语释义: [Kaikki] "прозрачная бесцветная жидкость"
```

LLM 融合处理：
1. 提取所有语义点，去重合并
2. 中文已有语义 → 保留中文表述
3. 中文未覆盖、英文/俄语有的语义 → 翻译为中文，作为新义项追加
4. 语义矛盾的 → 以中文源为准
5. 多义义项用 1) 2) 3) 编号

输出：
```
1) 水
2) 水面，水域
3) 水流
4) 矿泉
5) 空话，废话
6) 水分，体液
```

### 2.3 置信度

每个义项附带置信度（内部用，不展示给用户）：
- 高：多个来源语义一致
- 中：两个来源一致
- 低：单一来源

### 2.4 LLM Prompt

```
你是俄中词典编纂专家。以下是词条「{lemma}」({pos}) 的多源释义数据。
请融合为一条最完整的中文释义。

中文释义: {bkrs_zh}
英文翻译: {openrussian_en}
英文释义: {kaikki_en}
俄语释义: {kaikki_ru}

要求:
1. 以中文释义为主干，保留原有义项编号和释义内容
2. 英文/俄语中有而中文未覆盖的语义点，翻译为中文，追加为新义项
3. 语义重复的合并为一条
4. 语义矛盾的以中文为准
5. 多义项用 1) 2) 3) 编号
6. 不标注来源
7. 释义简洁准确，每个义项一行
8. 保留 BKRS 释义中的例句搭配 (如 "• морская вода - 海水")

输出JSON:
{
  "definition": "融合后的中文释义",
  "confidence": 3,
  "new_senses_count": 2
}
```

## 三、变格变位融合规则

### 3.1 核心原则

所有来源的变格变位数据合并为一张表，按 (word_id, form) 去重，grammar_tag 规范化为统一格式。

### 3.2 来源优先级

当一个词条的同一变格形式在多个来源中出现时，grammar_tag 按来源优先级取最优值：

```
OpenRussian (列名直接映射，最规范) > Kaikki (tag需清洗) > pymorphy3生成 (兜底)
```

### 3.3 grammar_tag 规范化

所有 tag 统一为 `{case}_{number}` 或 `{tense}_{person}_{number}` 格式。

#### 名词 — 12行 (6格 × 单复数)

| 规范tag | 含义 | OpenRussian列名 | Kaikki tag匹配 |
|---------|------|-----------------|----------------|
| nom_sg | 主格单数 | sg_nom | nominative + singular |
| gen_sg | 生格单数 | sg_gen | genitive + singular |
| dat_sg | 与格单数 | sg_dat | dative + singular |
| acc_sg | 宾格单数 | sg_acc | accusative + singular |
| ins_sg | 工格单数 | sg_inst | instrumental + singular |
| prep_sg | 前置格单数 | sg_prep | prepositional + singular |
| nom_pl | 主格复数 | pl_nom | nominative + plural |
| gen_pl | 生格复数 | pl_gen | genitive + plural |
| dat_pl | 与格复数 | pl_dat | dative + plural |
| acc_pl | 宾格复数 | pl_acc | accusative + plural |
| ins_pl | 工格复数 | pl_inst | instrumental + plural |
| prep_pl | 前置格复数 | pl_prep | prepositional + plural |

#### 动词 — 全变位

| 规范tag | 含义 | OpenRussian列名 |
|---------|------|-----------------|
| pres_1sg | 现在时第一人称单数 | presfut_sg1 |
| pres_2sg | 现在时第二人称单数 | presfut_sg2 |
| pres_3sg | 现在时第三人称单数 | presfut_sg3 |
| pres_1pl | 现在时第一人称复数 | presfut_pl1 |
| pres_2pl | 现在时第二人称复数 | presfut_pl2 |
| pres_3pl | 现在时第三人称复数 | presfut_pl3 |
| past_m | 过去时阳性 | past_m |
| past_f | 过去时阴性 | past_f |
| past_n | 过去时中性 | past_n |
| past_pl | 过去时复数 | past_pl |
| imp_sg | 命令式单数 | imperative_sg |
| imp_pl | 命令式复数 | imperative_pl |

#### 形容词 — 24行 (6格 × 4性/数)

| 规范tag | 含义 | OpenRussian列名 |
|---------|------|-----------------|
| nom_m | 主格阳性 | decl_m_nom |
| nom_f | 主格阴性 | decl_f_nom |
| nom_n | 主格中性 | decl_n_nom |
| nom_pl | 主格复数 | decl_pl_nom |
| gen_m | 生格阳性 | decl_m_gen |
| ... | (同上6格×4性/数) | ... |

### 3.4 噪声清洗

删除 inflections 中 grammar_tag 包含以下关键词的行：

```
romanization          — 罗马化转写，不是真实词形
ru-conj               — 模板标记
ru-noun-table         — 模板标记
inflection-template   — 模板标记
hard-stem             — 类标记，不是变格形式
accent-d              — 重音类型标记
class                 — 类标记
noun-from-verb        — 派生词标记
```

### 3.5 去重规则

```
同一 word_id + 同一 form → 合并为一行
- 如果两行 form 相同但 tag 不同 → 合并 tag (逗号分隔)
- 如果两行 form 和 tag 都相同 → 删除重复行
- form 去重音后比较 (воды́ 和 воды 视为同一个 form)
```

### 3.6 pymorphy3 兜底

对于 OpenRussian 和 Kaikki 都未覆盖的词条：
- 用 pymorphy3 解析 lemma，生成完整变格表
- tag 映射到规范格式
- 标记为生成数据（内部标记，不展示）

## 四、例句融合规则

### 4.1 核心原则

所有来源的例句合并为一张表，按句子去重，按质量排序，每个词条保留最多 5 条例句。

### 4.2 来源

| 来源 | 特点 | 预期量 |
|------|------|--------|
| Tatoeba | 中俄对齐，社区众包 | ~1.18万句对 |
| BKRS内嵌 | 释义中"• xxx - yyy"格式提取 | ~5万句对 |
| Kaikki | 英文例句(需翻译) | ~2万句 |
| 官媒(Phase 4) | 中俄双语新闻长句 | ~5-10万句对 |

### 4.3 BKRS内嵌例句提取规则

BKRS 释义文本中的例句格式：
```
• морская вода - 海水
• пресная вода - 淡水
```

提取正则：
```
•\s+([^-\n]+?)\s*-\s*([^\n•]+)
```

- 俄语部分 → sentence_ru
- 中文部分 → sentence_zh
- 来源标记 → 'BKRS'

### 4.4 去重规则

```
去重键 = sentence_ru 去除标点和空格后的小写字符串

如果两个例句的 sentence_ru 去标点去空格小写后相同:
  → 保留中文翻译更完整的一条
  → 或者保留来源优先级更高的一条
```

来源优先级（去重时保留）：
```
官媒 > Tatoeba > BKRS内嵌 > Kaikki
```

### 4.5 排序规则

每个词条的例句按以下规则排序后取前5条：

1. 含有该词条 lemma 原形的句子优先
2. 句子长度 10-40 词的优先（太短信息量不够，太长展示不下）
3. 有中文翻译的优先
4. 来源优先级高的优先

### 4.6 Kaikki英文例句处理

Kaikki 的例句是英文+俄语对，需要 LLM 翻译为中文：
- 俄语原句保留
- 英文翻译 → LLM 翻译为中文 → sentence_zh
- 批量处理，与释义融合分开执行

## 五、词性补全规则

### 5.1 优先级

```
pymorphy3 分析结果 > OpenRussian pos > Kaikki pos > BKRS pos > 后缀启发式
```

### 5.2 后缀启发式（兜底）

| 后缀 | 词性 |
|------|------|
| -ть | verb |
| -ся | verb (反身) |
| -ый / -ой / -ий | adj |
| -ая / -яя | adj (阴性) |
| -ое / -ее | adj (中性) |
| -ые / -ие | adj (复数) |
| -ый.*/-ой.* | adj |
| 其他默认 | noun |

### 5.3 目标

POS 空值率从 82% 降到 < 5%。

## 六、词频融合规则

### 6.1 来源

| 来源 | 数据 |
|------|------|
| OpenRussian | 词频排名 (1-10万) |
| Sharoff词频表 | 词频排名 (1-10万) |
| Kaikki | 无直接词频，但有 sense 频度标签 |

### 6.2 融合规则

```
如果 OpenRussian 有词频 → 用 OpenRussian 值
否则如果 Sharoff 有 → 用 Sharoff 值
否则 → NULL (后续可用 pymorphy3 频率统计估算)
```

### 6.3 难度等级映射

```
词频 1-2000     → A1
词频 2001-5000   → A2
词频 5001-10000  → B1
词频 10001-20000 → B2
词频 20001-50000 → C1
词频 50001+      → C2
无词频           → NULL
```

## 七、词源/同义词/反义词

- synonyms → word_stats.synonyms (逗号分隔)
- antonyms → word_stats.antonyms (逗号分隔)

## 九、质量验证标准

| 检查项 | 标准 | 验证方法 |
|--------|------|----------|
| 词条总数 | > 40万 | SELECT COUNT(*) FROM words |
| POS覆盖率 | > 95% | SELECT COUNT(*) FROM words WHERE pos IS NULL |
| 释义覆盖率 | 100% | 每个词条至少1条中文释义 |
| 变格完整率 | 名词=12行 | 抽查100个名词，验证6格×单复数 |
| 变格噪声率 | < 2% | 抽查100条，统计无效tag占比 |
| 例句覆盖 | 前1万高频词≥3条/词 | SELECT word_id, COUNT(*) FROM examples GROUP BY word_id |
| 去重 | 无重复例句 | sentence_ru去标点后DISTINCT |
| FTS5俄语搜索 | < 10ms | EXPLAIN QUERY PLAN + 计时 |
| 中文反搜 | < 500ms | 计时 |
| dict.db大小 | < 200MB | 文件大小 |
