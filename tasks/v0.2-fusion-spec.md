# Parus v0.2 — 全量多源语义融合规范

> 目标：将 BKRS + OpenRussian + Kaikki + Tatoeba 四源数据，通过 LLM 语义级理解，融合为每个俄语词条最完整的中文释义+例句+变格变位体系。
> 方案：方案A — 全量44万条词条都过 LLM。
> 本文档是执行规范，不是代码。代码实现按此规范编写。

## 一、数据源清单与角色定位

| 数据源 | 物理路径 | 格式 | 词条数 | 核心角色 |
|--------|----------|------|--------|----------|
| BKRS | app/src/main/assets/database/dict.db (已有) | SQLite | 255,188 | 中文释义主力 + 内嵌例句 |
| OpenRussian | ~/Downloads/russian-dictionary-master.zip | 4个CSV (nouns/verbs/adjectives/others) | 58,829 | 变格变位主力 + 英文翻译 + 词频 |
| Kaikki | ~/Downloads/kaikki.org-dictionary-Russian.jsonl | JSONL | 441,629 | 变格补充 + 词源 + 同义反义 + 英文释义 |
| Tatoeba | ~/Downloads/Sentence pairs in Russian-Mandarin Chinese - 2026-06-26.tsv | TSV | 11,803 | 中俄例句对 |
| pymorphy3 | Python环境 (已安装) | Python库 | 39万词内置 | 形态分析 + 变格生成 + 例句分词关联 |

### 各源数据结构（已校验）

#### BKRS (现有 dict.db)
- words(id, lemma, lemma_stressed, pos) — 255,188行，82%缺pos
- definitions(id, word_id, source, definition) — 255,248行，全source='BKRS'
- inflections(id, word_id, form, grammar_tag) — 998,867行，~30%噪声

#### OpenRussian CSV (解压后4个文件)
- nouns.csv: 26,983行，列含 bare/accented/translations_en/gender/sg_nom/sg_gen/.../pl_prep (21列)
- verbs.csv: 14,872行，列含 bare/accented/translations_en/aspect/imperative_sg/imperative_pl/past_m/past_f/past_n/past_pl/presfut_sg1/.../presfut_pl3 (18列)
- adjectives.csv: 11,942行，列含 bare/accented/translations_en/comparative/superlative/short_m/.../decl_pl_prep (34列)
- others.csv: 5,032行

#### Kaikki JSONL (每行一个JSON对象)
- word: 词条原形 (带重音)
- pos: 词性 (noun/verb/adj/adv/etc)
- forms: 数组，每个元素 {form, tags[]} — 变格变位形式
- senses: 数组，每个元素 {glosses[], translations[], examples[]} — 释义+例句
  - 注意：translations 中 code='zh' 的极少，glosses 是英文
- etymology_text: 词源文本

#### Tatoeba TSV
- 4列: 俄语句子ID / 俄语句子文本 / 中文句子ID / 中文句子文本
- 编码: UTF-8-sig (带BOM)

## 二、目标数据库 Schema (v2)

在 Antigravity 的 schema 基础上，增加 LLM 融合所需字段。

### words 表 (扩展)
```sql
CREATE TABLE words (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    lemma TEXT NOT NULL,              -- 去重音小写，用于搜索匹配
    lemma_stressed TEXT NOT NULL,     -- 带重音，用于UI展示
    pos TEXT,                         -- 词性 (noun/verb/adj/adv/...)
    frequency INTEGER,                -- 词频排名 (1-100000，越小越常用)
    conjugation_type INTEGER           -- 动词变位类型 (1=第一变位, 2=第二变位, 3=不规则, NULL=非动词)
);
```

### definitions 表 (扩展)
```sql
CREATE TABLE definitions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id INTEGER NOT NULL,
    source TEXT NOT NULL,              -- 'BKRS' / 'OpenRussian' / 'Kaikki' / 'AI-Fused' / 'Wiktionary-RU'
    definition TEXT NOT NULL,           -- 中文释义
    confidence INTEGER DEFAULT 0,       -- 置信度 0-3 (3=四源交叉验证, 2=三源, 1=两源, 0=单源)
    is_primary INTEGER DEFAULT 0,      -- 1=主释义(UI优先展示), 0=补充释义
    FOREIGN KEY(word_id) REFERENCES words(id) ON DELETE CASCADE
);
```

### inflections 表 (不变)
```sql
CREATE TABLE inflections (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id INTEGER NOT NULL,
    form TEXT NOT NULL,
    grammar_tag TEXT,                  -- 规范化标签，见第四节
    FOREIGN KEY(word_id) REFERENCES words(id) ON DELETE CASCADE
);
```

### examples 表 (新增)
```sql
CREATE TABLE examples (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id INTEGER NOT NULL,
    sentence_ru TEXT NOT NULL,
    sentence_zh TEXT,
    source TEXT NOT NULL,              -- 'Tatoeba' / 'BKRS-embedded' / 'Kaikki' / 'Sputnik' / 'TASS' / 'RT' / 'Kremlin'
    FOREIGN KEY(word_id) REFERENCES words(id) ON DELETE CASCADE
);
```

### word_stats 表 (新增)
```sql
CREATE TABLE word_stats (
    word_id INTEGER PRIMARY KEY,
    frequency_rank INTEGER,            -- OpenRussian/Sharoff 词频排名
    difficulty_level TEXT,             -- TRKI/HSK等级 (A1/A2/B1/B2/C1/C2)
    etymology TEXT,                    -- 词源 (来自Kaikki)
    synonyms TEXT,                     -- 同义词 (逗号分隔，来自Kaikki)
    antonyms TEXT,                      -- 反义词 (逗号分隔，来自Kaikki)
    FOREIGN KEY(word_id) REFERENCES words(id) ON DELETE CASCADE
);
```

### FTS5 虚拟表 (重建)
```sql
-- 俄语搜索 (FTS5 对西里尔字母分词正常)
CREATE VIRTUAL TABLE words_fts USING fts5(lemma, lemma_stressed, content='words', content_rowid='id');

-- 中文反搜 (LIKE+GLOB 6级排序，不用FTS5)
-- 中文搜索逻辑不变，保持 v0.1 的 WordDao.kt 实现
```

## 三、融合管线架构

### 总体流程
```
Phase 1: 数据预处理 (无LLM)
  BKRS SQLite → 解压 → lemma索引
  OpenRussian CSV → 解析 → 变格表 + 英文翻译 + 词频
  Kaikki JSONL → 流式解析 → 英文释义 + forms + 词源 + 同义反义
  Tatoeba TSV → 解析 → 句对
  pymorphy3 → 例句分词 → word_id关联
  ↓
  产出: intermediate.jsonl (每行一个词条，聚合所有源的数据)

Phase 2: LLM 语义融合 (核心)
  逐条读取 intermediate.jsonl
  ↓
  LLM 输入: {lemma, pos, BKRS中文释义, OpenRussian英文, Kaikki英文, Kaikki俄语释义}
  ↓
  LLM 输出: {fused_definition, confidence, new_definitions[], semantic_notes}
  ↓
  产出: fused.jsonl

Phase 3: 数据库构建 (无LLM)
  fused.jsonl + 变格表 + 例句表 → build_parus_db.py → dict.db (v2)
  + FTS5索引 + 质量报告
```

### Phase 1: 数据预处理详细规范

#### 1.1 BKRS 提取
- 从现有 dict.db 读取 words + definitions
- 建立 lemma → {id, pos, definitions[]} 索引
- 从 definitions.definition 中正则提取内嵌例句 (格式: `• xxx - yyy`)
- POS 缺失的标记为待补全

#### 1.2 OpenRussian 解析
- 解压 zip，读取4个CSV
- 每个CSV按词性(noun/verb/adj/other)解析
- 提取字段:
  - bare → lemma (去重音)
  - accented → lemma_stressed
  - translations_en → 英文释义列表
  - 变格列 → 规范化为 inflections 行 (见第四节)
  - gender/aspect/partner → 元数据
- 建立 lemma → {pos, en_translations, inflections[], frequency} 索引

#### 1.3 Kaikki 流式解析
- 890MB JSONL，逐行读取，不能全部加载到内存
- 每条提取:
  - word → lemma (去重音)
  - pos → 词性
  - forms[] → 变格变位 (需清洗: 去除romanization, template标记)
  - senses[].glosses → 英文释义列表
  - senses[].translations (code='zh') → 中文翻译 (极少但提取)
  - senses[].examples → 例句
  - etymology_text → 词源
  - synonyms/antonyms → 同义反义 (如果有)
- 建立 lemma → {pos, en_glosses[], forms[], etymology, synonyms, antonyms} 索引
- 去重: 同一 lemma 可能有多条JSONL行 (不同词性)，合并

#### 1.4 Tatoeba 例句关联
- 读取TSV (注意BOM)
- 每个俄语句子用 pymorphy3 分词 → 获取每个词的 normal_form
- 如果 normal_form 在 words 表中 → 插入 examples(word_id, sentence_ru, sentence_zh, source='Tatoeba')
- 一个句子可能关联多个 word_id (每个词一条记录)

#### 1.5 聚合产出
- 合并所有源的 lemma 集合 (BKRS 25万 + Kaikki 44万 + OpenRussian 5.8万)
- 每个 lemma 聚合所有源的数据
- 输出 intermediate.jsonl，每行一个词条:
```json
{
  "lemma": "вода",
  "lemma_stressed": "вода́",
  "pos_sources": {"BKRS": "noun", "Kaikki": "noun", "OpenRussian": "noun"},
  "bkrs_definitions": ["1) 水 shuǐ\n• морская вода - 海水\n..."],
  "openrussian_en": ["water", "aqua"],
  "kaikki_en": ["water", "moisture", "waterway", "body of water"],
  "kaikki_ru_glosses": ["прозрачная бесцветная жидкость"],
  "kaikki_zh": [],
  "etymology": "From Proto-Slavic *vodā",
  "synonyms": ["водичка", "водица"],
  "frequency": 42,
  "inflections": [{"form": "воды", "tag": "genitive_singular"}, ...],
  "examples": [{"ru": "Мне пора идти спать.", "zh": "我该去睡觉了。", "source": "Tatoeba"}]
}
```

### Phase 2: LLM 语义融合详细规范

#### 2.1 LLM 选择
- 主力: ark-code-latest (方舟Agent Plan Auto模式)
- 通过 Hermes API 调用 (本地 HTTP 请求)
- 批量处理: 每次请求处理 1 个词条 (保证质量)
- 预估: 44万条 × 单条约800 token = 3.5亿 token

#### 2.2 Prompt 模板
```
你是俄中词典编纂专家。下面是一个俄语词条的多源数据，请进行语义级融合，输出最完整的中文释义。

词条: {lemma} ({pos})
词源: {etymology}

数据源1 — BKRS俄汉词典:
{bkrs_definitions}

数据源2 — OpenRussian英文翻译:
{openrussian_en}

数据源3 — Kaikki/Wiktionary英文释义:
{kaikki_en}

数据源4 — Kaikki/Wiktionary俄语单语释义:
{kaikki_ru_glosses}

请输出JSON:
{
  "fused_definition": "融合后的中文释义(多义词用1)2)3)编号，每个语义点标注来源一致性)",
  "confidence": 1-3,
  "new_senses": [
    "BKRS未覆盖、从英文释义补充的新语义点(中文翻译)"
  ],
  "semantic_notes": "语义歧义/语境标注(如正式/口语/专业)",
  "pos_corrected": "修正后的词性(如果原pos为空或错误)"
}

规则:
1. BKRS中文释义为主干，不破坏原有语义
2. 英文释义中BKRS未覆盖的语义点→翻译为中文，加入new_senses
3. confidence: 3=四源语义一致, 2=三源一致, 1=仅两源, 0=单源无交叉
4. 不要合并不同语义点的释义，保持多义项编号结构
5. 如果英文释义与中文矛盾，以中文为准，英文记入semantic_notes
6. 输出必须是合法JSON，不要加markdown代码块标记
```

#### 2.3 处理策略

按词频分3批处理 (优先处理高频词):

| 批次 | 词条数 | 特征 | 处理方式 |
|------|--------|------|----------|
| 批次1 | 前5万 | 高频核心词，四源都有 | 全量LLM融合，最高质量 |
| 批次2 | 5-20万 | 中频词，BKRS+Kaikki重叠 | 全量LLM融合 |
| 批次3 | 20-44万 | 低频稀有词，仅Kaikki有 | 全量LLM翻译(英文→中文) |

每批处理完后:
- 输出进度到日志
- 检查输出JSON合法性 (json.loads 验证)
- 失败重试 (最多3次)
- 记录失败词条，最后人工检查

#### 2.4 输出格式 (fused.jsonl)
每行一个JSON:
```json
{
  "lemma": "вода",
  "lemma_stressed": "вода́",
  "pos": "noun",
  "fused_definition": "1) 水 shuǐ 〔BKRS+OpenRussian+Kaikki三源一致〕\n2) 水面，水域 shuǐmiàn 〔BKRS+Kaikki〕\n3) 水流 shuǐliú 〔BKRS〕\n4) 矿泉 kuàngquán 〔BKRS〕\n5) 空话，废话 kōnghuà 〔BKRS〕\n6) 水分，体液 〔Kaikki英文补充，AI翻译〕",
  "confidence": 3,
  "new_senses": ["水分，体液"],
  "semantic_notes": "много воды可指'废话多'(口语)或'水量多'(字面)，需语境区分",
  "definitions_to_insert": [
    {"source": "AI-Fused", "definition": "1) 水 shuǐ ... 6) 水分，体液", "confidence": 3, "is_primary": 1},
    {"source": "BKRS", "definition": "1) 水 shuǐ ...", "confidence": 1, "is_primary": 0},
    {"source": "AI-Enhanced", "definition": "水分，体液 (Kaikki英文补充)", "confidence": 1, "is_primary": 0}
  ]
}
```

## 四、变格变位 grammar_tag 规范化

### 问题
当前 inflections.grammar_tag 来自 Wiktionary，格式混乱:
- 模板标记: ru-conj, ru-noun-table
- 罗马化: romanization
- 类标记: hard-stem, accent-d', class
- 古旧: dated
- 真实标签格式不统一: nominative / nom / И. / first-person

### 规范化映射表

#### 名词变格
| 规范化tag | 匹配模式 (grammar_tag contains) |
|-----------|--------------------------------|
| nominative_singular | nominative, nom, И., singular, sg |
| genitive_singular | genitive, gen, Р., род, singular, sg |
| dative_singular | dative, dat, Д., дат, singular, sg |
| accusative_singular | accusative, acc, В., вин, singular, sg |
| instrumental_singular | instrumental, ins, Т., твор, singular, sg |
| prepositional_singular | prepositional, prep, П., предл, singular, sg |
| nominative_plural | nominative, nom, И., plural, pl |
| genitive_plural | genitive, gen, Р., род, plural, pl |
| ... (同上6格 × 复数) | |

#### 动词变位
| 规范化tag | 匹配模式 |
|-----------|----------|
| present_1sg | first-person, 1st, singular, presfut_sg1 |
| present_2sg | second-person, 2nd, singular, presfut_sg2 |
| present_3sg | third-person, 3rd, singular, presfut_sg3 |
| present_1pl | first-person, 1st, plural, presfut_pl1 |
| present_2pl | second-person, 2nd, plural, presfut_pl2 |
| present_3pl | third-person, 3rd, plural, presfut_pl3 |
| past_masculine | past, masculine, masc |
| past_feminine | past, feminine, fem |
| past_neuter | past, neuter, neut |
| past_plural | past, plural |
| imperative_singular | imperative, singular |
| imperative_plural | imperative, plural |

#### 形容词变格
| 规范化tag | 匹配模式 |
|-----------|----------|
| {case}_{gender} | nominative_masculine, genitive_feminine, etc |

### 清洗规则
1. 删除 grammar_tag 包含以下关键词的行: romanization, ru-conj, ru-noun-table, inflection-template, hard-stem, accent-d', class, noun-from-verb
2. 标记 dated 的行: 保留但 grammar_tag 追加 ";dated"
3. 去重: 同一 (word_id, form) 保留一条，合并 grammar_tag
4. OpenRussian 变格直接用列名映射 (sg_nom → nominative_singular)

## 五、官媒例句扩展规范 (Phase 4, 可选)

### 目标
从俄罗斯官媒中俄双语文章抓取长例句，补充 Tatoeba 短句的不足。

### 数据源
| 来源 | 俄文URL | 中文URL | 特点 |
|------|---------|---------|------|
| Sputnik | sputniknews.ru | cn.sputniknews.com | 新闻最全，更新快 |
| RT | rt.com/russian | rt.com/chinese | 国际新闻 |
| TASS | tass.ru | tass.com.cn | 官方通讯社 |
| Kremlin | kremlin.ru | cn.kremlin.ru | 总统讲话 |

### 抓取规范
1. 抓取频率: 每周一次，抓最近一周文章
2. 文章选择: 优先中俄对齐的文章 (URL模式匹配)
3. 句子对齐: 用俄中词对齐算法 (pymorphy3 + jieba 分词 + TF-IDF相似度)
4. 质量过滤:
   - 句子长度 10-50 词 (太短信息量不够，太长难展示)
   - 俄语句子必须包含 words 表中的 lemma
   - 中文翻译必须非空
5. 每个词最多保留 5 条例句 (按来源优先级: TASS > Kremlin > Sputnik > RT > Tatoeba)
6. source 字段标注来源

### Cron 定时任务
- 每周日 03:00 执行一次
- 抓取上周文章 → 对齐 → 入库 → 输出报告

## 七、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| LLM 输出非法JSON | 融合失败 | json.loads 验证 + 重试3次 + 失败记录 |
| Kaikki 890MB 解析慢 | Phase 1 耗时长 | 流式读取，不加载全文件 |
| LLM 额度不够 | Phase 2 中断 | 分批处理，每批1万条，可断点续传 |
| 变格清洗误删 | 丢失正确词形 | 清洗前备份，清洗后抽查100个词验证 |
| 官媒抓取被封IP | Phase 4 失败 | 限制频率，加 User-Agent，通过VPS代理 |
