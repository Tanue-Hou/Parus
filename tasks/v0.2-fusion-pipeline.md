# Parus v0.2 融合管线设计

## 架构

```
Phase 1: 数据预处理 (纯Python，无LLM)
  ├─ 1a: BKRS SQLite → intermediate (255k词条)
  ├─ 1b: OpenRussian CSV → intermediate (变格+词频+英文翻译)
  ├─ 1c: Kaikki JSONL 流式 → intermediate (变格+词源+英文释义+例句)
  ├─ 1d: Tatoeba TSV → intermediate (例句)
  └─ 1e: 聚合 → intermediate.jsonl (每条聚合所有源)

Phase 2: LLM 语义融合 (核心)
  ├─ 2a: 释义融合 (BKRS中文 + OpenRussian英文 + Kaikki英文/俄语 → 最优中文释义)
  ├─ 2b: 词性补全 (pymorphy3 > OpenRussian > Kaikki > 后缀)
  ├─ 2c: 例句融合 (Tatoeba + BKRS内嵌 + Kaikki → 去重排序)
  └─ 2d: 变格融合 (OpenRussian > Kaikki > pymorphy3 兜底)

Phase 3: 数据库构建
  ├─ 3a: 创建 dict.db v2 schema
  ├─ 3b: 批量插入 words + definitions + inflections + examples + word_stats
  ├─ 3c: 创建 FTS5 索引
  └─ 3d: 质量报告

Phase 4: 官媒例句 (可选，后续)
```

## 文件清单

| 文件 | 功能 | 说明 |
|------|------|------|
| `pipeline/phase1_preprocess.py` | 数据预处理 | 读取所有源，产出 intermediate.jsonl |
| `pipeline/phase2_fuse.py` | LLM 语义融合 | 读取 intermediate.jsonl，调用 LLM 融合释义 |
| `pipeline/phase3_build_db.py` | 数据库构建 | 读取 fused.jsonl + 变格/例句数据，构建 dict.db |
| `pipeline/utils.py` | 工具函数 | clean_stress, clean_dsl, grammar_tag 规范化等 |
| `pipeline/config.py` | 配置 | 所有路径、参数 |
