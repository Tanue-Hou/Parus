# Parus 例句增强融合方案 v2
# =============================
# 侯总要求：每词5-10句，短语+短句+长句混合，覆盖多义项
# 三层结构：短语(层1) + 完整句(层2) + 新闻句(层3)
# 不标注重音，词头保留高亮

## 数据源

### 层1：BKRS内嵌短语（现有，不过滤）
- 来源：BKRS释义中的 • 和 → 标记的短语固定搭配
- 数量：18,288词，~50,735条
- 特点：短（avg 25字），固定搭配，接格关系
- 处理：保留所有，不过滤长度

### 层2：完整句（Tatoeba + Kaikki，15-50字）
- Tatoeba：7,242 lemma已索引，日常对话，翻译质量好
- Kaikki：书面语，avg 96字（太长需截断）
- 过滤：只保留15-50字之间的完整句
- 处理：每个义项至少配1条完整句

### 层3：新闻句（TASS + RIA + Interfax + CGTN，最近1年）
- TASS RSS：100条/次，只有title无正文 → 需要从文章页面提取
- RIA Novosti RSS：89条/次，文章正文可提取（article__body）
- Interfax RSS：25条/次，description已有正文片段，页面也可提取
- CGTN Russian：需要确认RSS可用性
- 处理：每日抓取RSS → 提取文章正文 → 用pymorphy3建lemma索引
- 每个新闻句标注来源为 "News:TASS" / "News:RIA" / "News:Interfax"

### 层4：LLM生成（补缺口，仅A1-B2高频词）
- 仅对A1-B2（20,000词）中层1+2+3覆盖不足的词
- 每词生成2-3条现代例句（科技/生活/职场）
- 用DeepSeek API，batch 5词/请求
- 不标注重音

## 例句组装策略（每词5-10条）

1. 解析BKRS释义中的义项编号（1) 2) 3) ...）
2. 每个义项至少配：
   - 1条短语（层1）
   - 1条完整句（层2或层3）
3. 优先级：层1 > 层3(新闻) > 层2(Tatoeba) > 层2(Kaikki) > 层4(LLM)
4. 总例句数控制在5-10条
5. 如果某义项无语料匹配，跳过不补

## 质量过滤

保留：
- 所有BKRS内嵌短语（不过滤长度）
- 15-50字完整句
- 新闻句（不过滤长度，但优先选前3句导语）
- 超长新闻句截断到50字

丢弃：
- 空ru/空zh
- 重复句
- 含古俄语拼写（ъ结尾词）
- 含复合将来时（буду + 动词不定式）

## 重音处理
- 所有例句文本strip重音标记（' ` \u0301）
- 词头lemma保留重音标记（UI中buildStressedAnnotatedString高亮）

## Pipeline改动

### 新建 phase2b_examples.py
功能：
1. 读取fused.jsonl
2. 解析BKRS释义义项
3. 从各源收集例句
4. 按义项组装5-10条
5. strip重音标记
6. 输出更新后的fused.jsonl

### 修改 phase3_build_db.py
- BKRS-embedded来源不过滤
- 其他来源只过滤空ru/空zh和重复句
- 所有例句strip重音标记
