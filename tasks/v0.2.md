# Parus v0.2 — 词库增强计划

> 目标：从单源 BKRS 词典升级为多源融合、语义增强、例句完备的高质量俄汉词典
> P0 核心：词库后端是 App 的本质，这一轮全部围绕数据质量

---

## 现状分析

### 已有数据
| 表 | 行数 | 来源 | 问题 |
|----|------|------|------|
| words | 255,188 | BKRS | 82%词条缺 pos（词性）标注 |
| definitions | 255,248 | BKRS 单一来源 | 无第二词库验证，释义不可交叉校验 |
| inflections | 998,867 | Wiktionary | 含大量噪声（模板标记、罗马化、古旧拼写重复） |

### 核心短板
1. **词性缺失**：209,359 条（82%）没有 pos 标注，影响搜索排序和语法展示
2. **变格噪声**：inflections 混入 `ru-conj`/`ru-noun-table`（模板标记）、`romanization`（罗马化）、`hard-stem`（类标记）、`dated`（古旧拼写）等非真实词形
3. **无例句表**：BKRS 释义内嵌例句，无法独立检索和展示
4. **无词频数据**：无法区分常用词和生僻词
5. **单源风险**：全部释义来自 BKRS，无交叉校验

## P0 — 词库增强核心任务

### task1 — Schema 扩展与迁移
- 新增 `examples` 表：`id, word_id, sentence_ru, sentence_zh, source`
- 新增 `word_stats` 表：`word_id, frequency_rank, difficulty_level`
- words 表新增 `frequency` 列（INTEGER，1-100000 排名）
- words 表新增 `conjugation_type` 列（动词变位类型：1变位/2变位/不规则）
- definitions 表新增 `confidence` 列（INTEGER，多源交叉校验分数）
- Room version 升级到 2，写 Migration 脚本
- 验证：旧版数据完整迁移，新增表可正常读写

### task2 — 变格数据清洗
- 清除 inflections 表中的噪声数据：
  - 模板标记：`ru-conj`, `ru-noun-table`, `inflection-template`
  - 罗马化：`grammar_tag='romanization'` 的行
  - 类标记：`hard-stem`, `accent-d'`, `class` 等
  - 非词形：`noun-from-verb`, `adjective,relational`, `diminutive` 等派生词
- 拆分现代形式和古旧形式（`dated` 标记的单独标记，UI 可切换显示）
- 去重：同一 word_id + form 保留一条，合并 grammar_tag
- 预期清洗后数据量：~70-80万条（从99.8万降至）
- 验证：вода 的变格只有 6格×单复数=12 行（清洗前有20行含噪声）

### task3 — POS 词性补全
- 从 Wiktionary/Kaikki 导出的 POS 数据回填到 words.pos
- 规则兜底：通过后缀启发式判断（-ть=verb, -ый/-ой=adj, -ая=adj fem）
- 目标：pos 空值率从 82% 降到 <5%
- 验证：随机抽 100 条检查准确性

### task4 — OpenRussian 词频与例句合并
- 下载 OpenRussian 数据（15万词条）
- 合并 strategy：
  - frequency_rank → word_stats.frequency_rank
  - 例句 → examples 表
  - 翻译 → definitions 表（source='OpenRussian'，与 BKRS 并列）
  - POS/gender → 回填 words 表
- BKRS 有但 OpenRussian 没有的词条：保留 BKRS，frequency=NULL
- OpenRussian 有但 BKRS 没有的词条：新增，补充翻译
- 验证：говорить 有例句、有词频排名、有 BKRS+OpenRussian 双释义

### task5 — Wiktionary 俄语释义合并
- 下载 Kaikki Wiktionary Russian dump
- 提取俄语单语释义 → definitions 表（source='Wiktionary-RU'）
- 提取例句 → examples 表
- 提取标准变位/变格表 → 清洗后替换 inflections
- 交叉校验：BKRS 翻译是否与 Wiktionary 释义一致
- 验证：вода 有 3 个来源释义（BKRS中→俄 / OpenRussian中→俄 / Wiktionary俄语单语）

### task6 — 构建管线重构
- 重写 build_bkrs_db.py → build_parus_db.py（多源融合版）
- 管线步骤：下载数据源 → 解析 → 清洗 → 合并 → 去重 → 校验 → 导出 SQLite
- 支持 --source 参数：可单独重建某源数据
- 支持 --validate 参数：自动质量检查（覆盖率、空值率、重复率）
- 输出构建报告：各源贡献词条数、合并冲突数、清洗删除数
- 验证：一键 `python build_parus_db.py --all` 生成完整 dict.db

## P1 — 做了明显更好

### task7 — AI 语义级释义增强
- 对 BKRS 释义过于简短的词条（<10字符），用 AI 生成补充释义
- 对高频前 5000 词，生成用法说明和搭配
- 对翻译歧义词条，标注语境（正式/口语/专业）
- 输出格式：source='AI-Enhanced'，标记 confidence=0.8（低于人工源）
- 验证：随机抽 50 条人工检查 AI 释义质量

### task8 — 例句独立建表与展示
- 从 BKRS 释义中解析提取例句（`• xxx - yyy` 格式）
- OpenRussian 和 Wiktionary 例句直接入表
- UI 改造：释义卡片下方独立展示例句区
- 验证：вода 有 5+ 条例句，可独立展示

### task9 — 变位表规范化展示
- 动词：6人称×3时态（现在/过去/将来）+ 命令式 + 形动词
- 名词：6格×单复数
- 形容词：6格×单复数×3性
- UI 改造：变位表格化展示（行=格/人称，列=数/时态）
- 验证：говорить 展示完整变位矩阵
